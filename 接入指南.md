# KsText 接入指南

这份文档不是给仓库维护者看的。

这份文档是给“想把 `KsText` 接到自己项目里的人”看的。

目标很简单：

- 让你的程序能读取 `KsText` 的包列表
- 让你的程序能搜索包
- 让你的程序能下载某个包的正文
- 让你的程序能把下载到的 killsay 文本实际用起来

如果你只想知道最少要接什么，先看这一段：

## 1. 最小接入流程

你的程序只要做这几步就能接：

1. 下载仓库根目录的 `index.json`
2. 读取里面的 `packs` 列表
3. 把 `name / author / summary / entryCount` 展示给用户
4. 用户选中某个包后，读取它的 `downloadUrl`
5. 再去下载这个 `downloadUrl` 指向的 JSON
6. 解析里面的 `entries`
7. 把 `entries` 当成你的 killsay 文本池使用

就这么多。

## 2. 仓库地址

仓库主页：

```text
https://github.com/bedkillerspacex-boop/KsText
```

当前默认分支：

```text
master
```

索引地址：

```text
https://raw.githubusercontent.com/bedkillerspacex-boop/KsText/master/index.json
```

如果你项目里允许用户自己填仓库地址，建议支持这几种输入：

- GitHub 仓库页地址
- `owner/repo` 短格式
- 直接填写 raw 的 `index.json` 地址

## 3. 你需要读哪些文件

仓库里对接入方最重要的只有两类文件：

```text
KsText/
├─ index.json
└─ packs/
   ├─ A.json
   ├─ B.json
   └─ C.json
```

含义：

- `index.json`
  - 包列表
  - 用来给前端搜索、展示、分页
- `packs/*.json`
  - 某一个具体文本包的正文
  - 真正要拿来给 killsay 用的内容在这里

## 4. `index.json` 结构

`index.json` 大概长这样：

```json
{
  "schemaVersion": 1,
  "generatedAt": "2026-05-26T00:00:00Z",
  "packs": [
    {
      "id": "情话版",
      "name": "情话版",
      "author": "佚名",
      "summary": "温和一点的文案包",
      "language": "zh-CN",
      "tags": ["killsay", "community"],
      "serverTags": ["generic"],
      "version": 1,
      "updatedAt": "2026-05-26T00:00:00Z",
      "entryCount": 18,
      "sha256": "xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx",
      "downloadUrl": "https://raw.githubusercontent.com/bedkillerspacex-boop/KsText/master/packs/%E6%83%85%E8%AF%9D%E7%89%88.json"
    }
  ]
}
```

### 你至少要用到的字段

- `id`
  - 唯一标识
  - 你可以拿它当内部选中值
- `name`
  - 给用户看的包名
- `author`
  - 给用户看的作者
- `summary`
  - 给用户看的简介
- `entryCount`
  - 给用户看的条数
- `downloadUrl`
  - 真正下载包正文的地址

### 其他字段有什么用

- `language`
  - 文案语言，目前一般是 `zh-CN`
- `tags`
  - 可用于搜索过滤
- `serverTags`
  - 可用于服务器分类
- `version`
  - 包版本号
- `updatedAt`
  - 最后更新时间
- `sha256`
  - 文件校验值

## 5. `packs/*.json` 结构

真正的文本包正文大概长这样：

```json
{
  "schemaVersion": 1,
  "id": "情话版",
  "name": "情话版",
  "author": "佚名",
  "version": 1,
  "updatedAt": "2026-05-26T00:00:00Z",
  "description": "温和一点的文案包",
  "language": "zh-CN",
  "tags": ["killsay", "community"],
  "serverTags": ["generic"],
  "entries": [
    "{name} 今晚先陪你到这里",
    "{name} 你的结局我先收下了",
    "{name} 风停了，你也该回去了"
  ]
}
```

### 你真正要拿来用的是

```json
"entries": [...]
```

这里面的每一条字符串，就是一条可发送的 killsay 文本。

## 6. 程序里怎么用 `entries`

最常见的用法有三种：

### 轮换发送

每次击杀就按顺序取一条：

1. 第一次取第 1 条
2. 第二次取第 2 条
3. 用完后回到第 1 条

### 随机发送

每次击杀从 `entries` 里随机抽一条。

### 模板替换

如果文本里带变量，比如：

```text
{name} 被我带走了
```

你就在发送前把 `{name}` 替换成目标玩家名。

## 7. 搜索应该怎么做

最简单的搜索逻辑：

对 `index.json` 里的每个包，匹配下面这些字段：

- `id`
- `name`
- `author`
- `summary`
- `tags`

只要有一个字段包含搜索词，就显示这个包。

## 8. 缓存应该怎么做

推荐你分两层缓存：

### 索引缓存

缓存 `index.json`

用途：

- 打开界面更快
- 网络失败时还能显示旧列表

### 包正文缓存

缓存 `packs/*.json`

用途：

- 用户重复选同一个包时不用每次都重新下载

## 9. 但是缓存要注意一个坑

只缓存，不失效，会出大问题。

最典型的问题就是：

1. 你同步了最新 `index.json`
2. 用户列表里看见了新版本信息
3. 但你本地还在读旧的 `packs/<id>.json`
4. 结果实际发送的还是旧文案

所以推荐做法：

- 同步索引成功后，清掉本地包缓存
- 或者至少清掉当前已选包的缓存
- 然后重新下载一次包正文

如果你不做这一步，就很容易出现“仓库更新了，但程序里还是旧文本”的问题。

## 10. 分支问题一定要统一

这个仓库当前默认按 `master` 来读。

所以你程序里如果自动拼 raw 地址，最好按这个分支来：

```text
https://raw.githubusercontent.com/bedkillerspacex-boop/KsText/master/index.json
```

以及：

```text
https://raw.githubusercontent.com/bedkillerspacex-boop/KsText/master/packs/xxx.json
```

不要一会儿拼 `main`，一会儿拼 `master`。

以前最容易出的问题就是：

- `index.json` 从 `master` 拉到了
- 但包正文地址拼成了 `main`
- 最后列表能显示，点包却 404

## 11. 推荐接入策略

如果你自己也在做一个 mod / 客户端 / 脚本工具，推荐这样接：

### 启动时

- 尝试读取本地缓存的 `index.json`
- 如果有缓存，先展示缓存内容
- 后台再去同步远端

### 打开云文案页时

- 用 `index.json` 生成列表
- 支持搜索
- 支持显示当前已选包

### 用户点击包时

- 先看本地是否已有这个包缓存
- 如果没有，下载 `downloadUrl`
- 如果有，也建议在最新索引同步后重新拉一次

### 用户点击保存时

- 把这个包的 `entries` 写入你自己的本地文本池
- 刷新你自己项目里的 killsay 编辑框或内存状态

## 12. 推荐错误提示

你至少应该区分这些错误：

- `Index is not JSON`
  - 索引地址不是 JSON
- `HTTP 404`
  - 地址错了
- `HTTP 403`
  - 访问被拦了
- `Load pack failed`
  - 具体包正文拉取失败
- `No index URL`
  - 用户没填地址

别只弹一个“失败了”。

错误越具体，越好排查。

## 13. 推荐兼容做法

如果你想对用户更友好，建议兼容这些输入：

### 仓库主页地址

```text
https://github.com/bedkillerspacex-boop/KsText
```

### 短格式

```text
bedkillerspacex-boop/KsText
```

### 直接 raw 地址

```text
https://raw.githubusercontent.com/bedkillerspacex-boop/KsText/master/index.json
```

程序内部统一转成可请求的 raw `index.json` 地址就行。

## 14. 最少示例伪代码

```text
index = GET(index.json)
packs = index["packs"]

show_list(packs)

selected = user_choose_pack()
pack_json = GET(selected["downloadUrl"])
entries = pack_json["entries"]

message = entries[next_index]
message = message.replace("{name}", player_name)
send_chat(message)
```

## 15. 一句话总结

接这个仓库，本质上就两步：

1. 先读 `index.json` 做列表
2. 再按 `downloadUrl` 读具体包正文

只要你把“索引”和“正文”分开处理，再把缓存失效做对，`KsText` 就很容易接进任何项目里。
